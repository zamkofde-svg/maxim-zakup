# Деплой на Render.com

Бесплатный план Render → публичная https-ссылка для заказчика.

## Шаг 1. Регистрация на Render (5 минут)

1. Открой https://render.com → **Sign Up** → войти через **GitHub** (тем же аккаунтом zamkofde-svg)
2. Подтвердить email
3. На Dashboard → **New +** → **Web Service**
4. **Connect a repository** → выбрать `zamkofde-svg/maxim-zakup`
5. Render автоматически прочитает `render.yaml` из репо и создаст сервис

## Шаг 2. Прописать переменные окружения

В созданном сервисе → **Environment** → **Add Environment Variable**:

### Переменная 1: `OPENROUTER_API_KEY`
Значение: твой OpenRouter ключ (тот что в `~/.config/maxim-zakup/openrouter.env`).
**Лучше создать новый** через openrouter.ai/keys (старый засветился в чате).

### Переменная 2: `GOOGLE_SA_JSON_CONTENT`
Значение: **содержимое** файла `~/.config/maxim-zakup/sa.json` одной строкой
(скопировать весь JSON, включая `{}`).

```bash
# чтобы получить значение для копирования:
cat ~/.config/maxim-zakup/sa.json
```

**Лучше также ротировать** Service Account ключ (он тоже засветился в чате):
- IAM & Admin → Service Accounts → maxim-zakup-bot → Keys
- Удалить старый, создать новый, его содержимое и вставить в env

## Шаг 3. Деплой

После сохранения env-переменных Render автоматически запустит билд.
Это займёт ~3–5 минут (`pip install` всех зависимостей).

Когда увидишь зелёный статус **Live** → открой ссылку (она будет вида
`https://maxim-zakup.onrender.com`).

При первом открытии БД пустая → автоматически запустится sync из Drive.
Подожди ~30 секунд, обнови страницу — данные появятся.

## Шаг 4. Поделиться с заказчиком

Эту же ссылку отправляешь заказчику. Никаких других действий не нужно.

## Особенности бесплатного плана

- **Засыпает после 15 минут без запросов.** Просыпается за 30–50 секунд
  при первом запросе. Заказчику можно так и сказать: «первое открытие в
  день — подождать минуту».
- **750 часов/мес** — этого хватит на полноценную работу.
- **БД не персистентная** — при каждом перезапуске SQLite пересоздаётся
  и заново синхронизируется из Drive. Это даже лучше — всегда свежие данные.
  Если когда-то понадобится сохранять причины отклонений между деплоями —
  поменяем на Postgres (Render даёт бесплатный 90-дневный Postgres).

## Если что-то пошло не так

В Render → твой сервис → **Logs** — там видно билд-логи и runtime-логи.
Скинь скриншот ошибки в чат — починим.
